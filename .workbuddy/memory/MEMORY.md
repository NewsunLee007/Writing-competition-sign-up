# 瑞安市英语写作大赛报名系统 - 项目记忆

## 项目概览
项目名称：瑞安市英语写作大赛报名系统
创建日期：2026年3月30日
技术栈：React + Vite + TailwindCSS (前端)，Express.js + PostgreSQL (后端)
部署平台：GitHub + Vercel + Neon PostgreSQL

## 架构设计
1. **前端**：React单页应用，响应式设计，支持手机和电脑访问
2. **后端**：RESTful API，JWT认证，PostgreSQL数据库
3. **存储**：Neon PostgreSQL（云端PostgreSQL服务）
4. **部署**：Vercel托管前端和Serverless后端函数
5. **自动化**：GitHub Actions CI/CD流水线

## 核心功能
1. **用户认证**：注册、登录、邮箱验证、密码重置
2. **报名管理**：填写报名信息、上传材料、报名状态跟踪
3. **准考证生成**：自动生成和下载准考证
4. **成绩管理**：成绩录入和查询系统
5. **后台管理**：管理员审核、数据统计、导出功能
6. **多端兼容**：响应式设计，适配手机、平板、电脑

## 技术实现要点
1. **前端**：
   - 使用TypeScript提高代码质量
   - TailwindCSS实现响应式设计
   - React Router处理路由

2. **后端**：

3. **数据库**：

4. **安全措施**：

   - 密码哈希存储
   - 输入验证和SQL注入防护

## 最新项目记忆（2026-03-31）

### 当前产品定位
- 系统名称：瑞安市第三届初中学生英语创意写作评审活动报名系统
- 产品定位：正式、清晰、低干扰的报名系统，不做宣传页式炫技设计
- 核心页面：首页、报名中心、准考证下载中心、管理员中心
- 核心任务：在线报名、Excel批量导入、准考证下载、管理员汇总导出与清空数据

### 已确认的长期业务规则
- 报名对象为瑞安市八年级学生
- 报名方式分为两类：学区推荐、直属学校
- 瑞中附初名额固定为 6，其他名额按已确认表执行
- 准考证号规则为：年份后两位 + 考场号两位 + 座位号两位，共 6 位
- 电话字段统一为中国大陆 11 位手机号校验
- 同一 IP 且同一学校或同一学区下，近期提交记录需要可回看
- 管理员具备查看进度、导出 Excel、导出准考证、清空报名数据能力

### 已确认的界面审美偏好
- 风格关键词：简介、清楚、克制、正式、清新、少 AI 味
- 避免事项：大段营销文案、过度装饰、明显 AI 腔、宣传海报感、信息拥挤
- 首页文案：允许中英混排，但必须自然，不要生硬“AI slogan”
- 版式倾向：信息秩序优先，先讲操作路径，再讲视觉
- 视觉语言：浅色系、米白、灰蓝、低饱和深蓝，小圆角但不过分软糯
- 卡片表现：边界清楚、层级明确、阴影轻，不做重装饰玻璃感
- 标题策略：主标题可以有气质，但功能区必须朴素直接
- 按钮策略：主按钮明确，次按钮收敛，入口不要太多太散

### 已确认的交互偏好
- 报名流程必须顺序清楚：先分类，再归属，再录入，再提交
- “添加学生”必须在信息录入区下面，录完即可继续追加
- 批量导入必须支持模板下载、上传预览、准确报错行号、忽略空白行
- 新手导引必须自动滚动到目标区域，不能让用户自己找
- 新手导引默认只在同一 IP 第一次进入报名中心时自动出现，后续可手动重新打开
- 新手导引控件应简洁，图标优先，避免大段固定提示条占空间

### 已确认的票证与下载偏好
- 准考证必须为中文语境，不使用明显英文票证风格
- 视觉风格应接近正式考试证件，而不是宣传卡片
- 一张 A4 放两位学生准考证，且不能粗暴压缩变形
- 批量 PDF 体积要控制，优先保证清晰度，再做适度压缩
- 普通用户除单条查询外，也要支持按学区或学校批量下载准考证

### 后续设计与开发必须遵循的约束
- 优先删除冗余块，而不是新增说明块
- 优先做“操作变清楚”，再做“样式变漂亮”
- 所有引导和提示都应帮助电脑不熟练的老师快速完成任务
- 如需新增页面元素，先判断是否真的减少操作负担
- 管理员敏感信息不得在页面上直接暴露

### 可复用的网页设计提示词
#### 提示词 A：系统首页改版
请为一个正式的教育类报名系统设计首页。整体气质要求简介、清楚、克制、正式、清新、少 AI 味，不要做成宣传海报页。配色使用米白、灰蓝、低饱和深蓝，视觉轻但有秩序。允许中英混排，但文案必须自然。信息结构上优先突出操作入口和通知内容，删除营销化板块，减少装饰元素。标题可以有气质，但功能区要朴素直接。卡片边界清楚、阴影轻、文字层级明确。不要使用夸张渐变、玻璃拟态、大面积口号文案。

#### 提示词 B：表单与报名流程改版
请把报名页面设计成一个让中小学老师容易上手的正式报名界面。重点不是炫技，而是让操作路径非常清楚：先选报名类别，再选归属，再填写学生，再提交。支持手工录入和 Excel 批量导入两种方式。界面要求低学习成本、低视觉噪音、少 AI 味。按钮入口明确，次级操作收敛。引导文案短而准，错误提示要具体。所有新增元素都要围绕“减少来回跳转、减少理解成本”展开。

#### 提示词 C：新手引导设计

#### 提示词 D：准考证设计
